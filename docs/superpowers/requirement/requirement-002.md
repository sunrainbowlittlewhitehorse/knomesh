### 一、问题描述
**根据问题描述修复，并且填充该文档中以下几点的内容**
1. 没有标签管理功能，需要实现标签管理功能，并添加标签管理页面。

### 二、修复过程
1. **服务层**：在 `services/tag_service.py` 中添加 `rename_tag(tag_id, new_name)` 函数，支持标签重命名。
2. **API 层**：在 `app.py` 中新增：
   - `GET /tags` — 标签管理页面路由，传入标签列表和 `active_page='tags'`
   - `PUT /api/tags/<id>` — 标签重命名 API 端点
3. **前端页面**：创建 `templates/tags.html`，包含：
   - 新建标签表单（输入框 + 创建按钮）
   - 标签列表网格（显示颜色圆点、标签名、关联项目数）
   - 重命名功能（点击编辑按钮进入内联编辑模式，确认或取消）
   - 删除功能（删除前弹窗确认）
   - 空状态提示
4. **导航**：将 `base.html` 侧边栏的"标签"从静态 `<span>` 改为可点击的 `<a href="/tags">`，并添加 `active_page == 'tags'` 条件控制高亮状态。
5. **交互逻辑**：创建 `static/js/tags.js`，处理标签的创建、重命名和删除操作，所有操作通过 API 异步完成。

### 三、修复范围
- `services/tag_service.py`：新增 `rename_tag()` 函数
- `app.py`：新增 `GET /tags` 路由和 `PUT /api/tags/<id>` API 端点
- `templates/base.html`：侧边栏"标签"改为可点击链接，支持 active 高亮
- `templates/tags.html`：**新文件**，标签管理页面模板
- `static/js/tags.js`：**新文件**，标签管理页面的交互逻辑

### 四、修复结果
- 新增独立的标签管理页面，可通过侧边栏导航访问
- 支持新建标签（回车或点击按钮）
- 支持重命名标签（内联编辑，支持确认/取消，空值或不修改自动退出编辑）
- 支持删除标签（弹窗确认，同步移除关联关系）
- 侧边栏"标签"链接支持当前页面高亮状态
